Introduction

ArcanPHP is a modern, PHP-based framework designed to provide developers with a powerful and intuitive toolkit for building web applications. Inspired by the elegance and simplicity of Laravel, ArcanPHP brings a fresh perspective to PHP development, combining best practices with a flexible architecture to streamline the development process. Whether you're building small projects or complex enterprise-level applications, ArcanPHP offers a robust set of features that cater to developers of all skill levels. With a strong focus on developer experience, ArcanPHP includes built-in tools for routing, templating, ORM, and more, all designed to help you write clean, maintainable code. .

Why ArcanePHP

Simplicity and Flexibility: ArcanePHP is designed with a focus on simplicity and flexibility. It offers a clean, intuitive syntax that allows developers to build robust applications quickly and efficiently, without sacrificing control over the underlying code.

Modern PHP Practices: Built on the latest PHP standards, ArcanePHP embraces modern development practices such as dependency injection, service containers, and expressive routing. This ensures that your applications are not only easy to maintain but also future-proof.

Powerful Tooling: ArcanePHP comes with a rich set of built-in tools and features that streamline common development tasks. From an elegant ORM for database interactions to a powerful templating engine, ArcanePHP provides everything you need to develop full-featured applications out of the box.

Community-Driven: Like its inspiration, ArcanePHP is built with community contributions in mind. It’s designed to be extendable, allowing developers to easily create and share packages that can be seamlessly integrated into any ArcanePHP project.

Lightweight Performance: Despite its rich feature set, ArcanePHP remains lightweight and optimized for performance. Its modular architecture ensures that you only use the components you need, reducing overhead and improving application speed.

Installation

Before creating a acranePHP project ,Make sure your system have Php ,Git, Xampp and Composer installed.

After you have installed xampp ,you may clone new project instance from GIT from your terminal: 

    user@os:~/php/projects/location/htdocs$ git clone git@github.com:ArcanePHP/ArcanePHP.git my-project

Once you have started the xampp server, your application will be accessible

in your web browser at http://localhost/my-project/.

Initial configuration

The initial configuration of the ArcanePHP project lays the foundation for a smooth development experience. After cloning the repository, the first step involves setting up the environment variables by copying the .env.example file to .env and updating it with your specific configuration details, such as database credentials and application keys.

However, you may wish to review the App/Config.php file and its documentation also

Database and migration

you can update your .env configuration file to use the appropriate database. For example, if you wish to use MySQL, update your .env configuration file's DB_* variables like so:


	DB_CONNECTION=mysql
	DB_HOST=localhost
	DB_DATABASE=my-project
	DB_USERNAME=root
	DB_PASSWORD=

Essentials

Routing

ArcanePHP's Router provides an intuitive and straightforward way to define routes, allowing developers to map URIs to specific behaviors within their application with minimal configuration. With ArcanePHP, routing is designed to be simple yet powerful, making it easy to manage your application's endpoints.

You can define routes by specifying a URI and a closure, which handles the request. This approach ensures that your routes are both easy to read and maintain, without the need for extensive routing configuration files.

Named routes

Named routes provide a convenient way to generate URLs or redirect to specific routes, enhancing the readability and maintainability of your code.

ArcanePHP allows you to assign names to your routes, making it easier to reference them elsewhere in your application. This is especially useful when generating URLs or performing redirects.

Always use underscore instead of spaces while naming a route.

Grouped Routes

Route groups allow you to apply common attributes, such as middleware or prefixes, to multiple routes at once, simplifying route management.

ArcanePHP supports grouping routes to share common configurations. You can apply middleware, namespaces, or URL prefixes to a group of routes, making it easier to manage related routes in your application.
